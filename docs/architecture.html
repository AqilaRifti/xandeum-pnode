Architecture

Xandeum pNode Analytics follows a feature-based architecture pattern optimized for monitoring decentralized storage nodes.

Data Flow

The application follows a server-first data flow pattern:

  1. Data Source - pNode data stored in pnodes.json
  2. Server Functions - nodeData.server.ts processes data
  3. Server Components - Fetch and render data on server
  4. Client Components - Interactive features and real-time updates
// src/app/page.tsx - Main entry point
import { DashboardClient } from '@/components/dashboard/DashboardClient';
import { getDashboardData } from '@/lib/nodeData.server';

export default async function Home() {
  const data = await getDashboardData();
  return <DashboardClient initialData={data} />;
}

pNode Data Structure

Each pNode in the network has the following structure:

interface PNode {
  pubkey: string;           // Unique node identifier
  status: 'online' | 'offline';
  version: string;          // e.g., "0.8.0"
  storageUsed: number;      // Bytes
  storageTotal: number;     // Bytes
  storageCommitted: number; // Bytes
  storageUsagePercent: number;
  uptime: number;           // Days
  ip: string;               // "IP:port"
  address: string;
  isPublic: boolean;
  rpcPort: number;
  lastSeen: string;         // ISO timestamp
  lastSeenTimestamp: number;
  ramUsed?: number;         // Optional
  ramTotal?: number;        // Optional
}

Server-Side Data Processing

The nodeData.server.ts module handles data processing:

// lib/nodeData.server.ts
import pnodesData from '../../pnodes.json';

export async function getDashboardData() {
  const nodes = pnodesData as PNode[];
  
  return {
    nodes,
    stats: calculateStats(nodes),
    versionDistribution: getVersionDistribution(nodes),
    storageMetrics: getStorageMetrics(nodes),
  };
}

function calculateStats(nodes: PNode[]) {
  return {
    total: nodes.length,
    online: nodes.filter(n => n.status === 'online').length,
    offline: nodes.filter(n => n.status === 'offline').length,
    totalStorage: nodes.reduce((sum, n) => sum + n.storageTotal, 0),
    usedStorage: nodes.reduce((sum, n) => sum + n.storageUsed, 0),
  };
}

Geographic Data

The geo.server.ts module handles IP geolocation:

// lib/geo.server.ts
export async function getNodeLocations(nodes: PNode[]) {
  // Extract IP addresses and resolve to coordinates
  const locations = await Promise.all(
    nodes.map(async (node) => {
      const ip = node.ip.split(':')[0];
      const coords = await geolocateIP(ip);
      return { ...node, ...coords };
    })
  );
  return locations;
}

import { useState } from 'react';

import { Charts } from './Charts';

export function DashboardClient({ initialData }) {
  const [filters, setFilters] = useState({});
  
  const filteredNodes = applyFilters(initialData.nodes, filters);
  
  return (
    <div className="space-y-6">

}

State Management

Server State

  • Server Components - Data fetched at request time
  • Nuqs - URL-based state for filters and pagination

Client State

  • Zustand - Global client state (kanban, preferences)
  • React State - Local component state
// Using Nuqs for URL state
import { useQueryState } from 'nuqs';

function NodeFilters() {
  const [status, setStatus] = useQueryState('status');
  const [version, setVersion] = useQueryState('version');

        <SelectItem value="online">Online</SelectItem>
        <SelectItem value="offline">Offline</SelectItem>
      </Select>
    </div>
  );
}
