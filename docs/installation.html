Installation

Follow these steps to set up the Next.js Shadcn Dashboard Starter on your local machine.

Prerequisites

Before you begin, ensure you have the following installed:

  • Node.js 18+ - Download Node.js
  • Bun (recommended) or npm/yarn - Install Bun
  • Git - Download Git

Step 1: Clone the Repository

Clone the repository to your local machine:

git clone https://github.com/Kiranism/next-shadcn-dashboard-starter.git
cd next-shadcn-dashboard-starter

Step 2: Install Dependencies

Install the project dependencies using your preferred package manager:

# Using Bun (recommended)
bun install

# Using npm
npm install

# Using yarn
yarn install

# Using pnpm
pnpm install

Step 3: Environment Setup

Create a .env.local file by copying the example:

cp env.example.txt .env.local

Open .env.local and configure the required environment variables:

# Clerk Authentication (Required)
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_your_key_here
CLERK_SECRET_KEY=sk_test_your_key_here

# Clerk URLs (Optional - defaults work for most cases)
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up

# Sentry Error Tracking (Optional)
SENTRY_DSN=https://your_sentry_dsn_here
Important

You must set up Clerk authentication for the dashboard to work. See the Configuration page for detailed Clerk setup instructions.

Step 4: Start Development Server

Run the development server:

# Using Bun
bun run dev

# Using npm
npm run dev

# Using yarn
yarn dev

Open http://localhost:3000 in your browser to see the dashboard.

Success!

You should now see the dashboard running locally. Sign in with Clerk to access all features.

Available Scripts

Command Description
bun run dev Start development server
bun run build Build for production
bun run start Start production server
bun run lint Run ESLint
bun run format Format code with Prettier

Troubleshooting

Port Already in Use

If port 3000 is already in use, you can specify a different port:

bun run dev -- -p 3001

Dependency Issues

If you encounter dependency issues, try clearing the cache:

# Remove node_modules and lock file
rm -rf node_modules bun.lock

# Reinstall dependencies
bun install

Clerk Authentication Errors

If you see authentication errors:

  • Verify your Clerk API keys are correct
  • Ensure the keys match your Clerk application environment (development/production)
  • Check that your Clerk application has the correct redirect URLs configured
